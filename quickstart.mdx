## Implement Verification

Get up and running with Aqua Protocol's verification system in three simple steps. This guide will walk you through setting up your environment, creating your first verification, and integrating with our API.

<Info>
This quickstart assumes you have basic knowledge of blockchain concepts and JavaScript/TypeScript. If you're new to verification systems, check out our [Core Concepts](/essentials/settings) guide first.
</Info>

### Step 1: Set Up Your Environment

<AccordionGroup>
  <Accordion icon="download" title="Install the SDK">
    Install the Aqua Protocol SDK for verification and notarization:
    
    <CodeGroup>
    ```bash npm
    npm install @aqua-protocol/verifier
    ```
    
    ```bash yarn
    yarn add @aqua-protocol/verifier
    ```
    
    ```bash pnpm
    pnpm add @aqua-protocol/verifier
    ```
    </CodeGroup>
    
    <Check>
    The SDK includes TypeScript definitions and comprehensive verification tools.
    </Check>
  </Accordion>
  
  <Accordion icon="key" title="Get your API credentials">
    1. Sign up for a free account at [app.aqua-protocol.org](https://app.aqua-protocol.org)
    2. Navigate to the API Keys section in your dashboard
    3. Generate a new API key for verification services
    4. Copy your API key and verification endpoint
    
    <Warning>
    Keep your API key secure and never commit it to version control. Use environment variables in production.
    </Warning>
  </Accordion>
</AccordionGroup>

### Step 2: Create Your First Verification

<AccordionGroup>
  <Accordion icon="file-check" title="Define verification schema">
    Create a verification schema for document notarization:
    
    ```json schema.json
    {
      "type": "object",
      "properties": {
        "documentId": {
          "type": "string",
          "description": "Unique document identifier"
        },
        "hash": {
          "type": "string",
          "description": "SHA-256 hash of document content"
        },
        "timestamp": {
          "type": "string",
          "format": "date-time"
        },
        "signer": {
          "type": "string",
          "description": "Public key of document signer"
        }
      },
      "required": ["documentId", "hash", "timestamp", "signer"]
    }
    ```
    
    <Tip>
    This schema defines the structure for document verification and notarization.
    </Tip>
  </Accordion>
  
  <Accordion icon="rocket" title="Implement verification">
    Create and verify a document using the Aqua Protocol SDK:
    
    ```javascript verify.js
    import { AquaVerifier } from '@aqua-protocol/verifier';
    import crypto from 'crypto';
    
    const verifier = new AquaVerifier({
      apiKey: process.env.AQUA_API_KEY,
      endpoint: 'https://api.aqua-protocol.org/v3'
    });
    
    // Create document hash
    const documentContent = 'Important document content';
    const documentHash = crypto.createHash('sha256')
      .update(documentContent)
      .digest('hex');
    
    // Create verification
    const verification = await verifier.createVerification({
      documentId: 'doc-123',
      hash: documentHash,
      timestamp: new Date().toISOString(),
      signer: '0x742d35Cc6634C0532925a3b8D4C9db96C4b4d8b6'
    });
    
    console.log('Verification created:', verification.id);
    console.log('Transaction hash:', verification.txHash);
    ```
    
    <Check>
    Your document is now verified and notarized on the blockchain!
    </Check>
  </Accordion>
</AccordionGroup>

### Step 3: Query Verification Status

<AccordionGroup>
  <Accordion icon="search" title="Check verification status">
    Query your verification to confirm its status and details:
    
    ```javascript query.js
    // Get verification details
    const verification = await verifier.getVerification(verification.id);
    
    console.log('Verification Status:', verification.status);
    console.log('Document ID:', verification.documentId);
    console.log('Created At:', verification.createdAt);
    console.log('Block Number:', verification.blockNumber);
    ```
  </Accordion>
  
  <Accordion icon="check-circle" title="Verify document integrity">
    Verify that a document matches its notarized hash:
    
    ```javascript validate.js
    // Verify document against notarized hash
    const isValid = await verifier.validateDocument({
      documentId: 'doc-123',
      content: documentContent,
      expectedHash: verification.hash
    });
    
    if (isValid) {
      console.log('Document integrity verified');
    } else {
      console.log('Document has been modified');
    }
    ```
    
    <Warning>
    Always verify document integrity before trusting verification results in production applications.
    </Warning>
  </Accordion>
</AccordionGroup>
