# Getting Started with Aqua Protocol SDK

## Installation

```bash
npm install aqua-js-sdk
```

## Quick Start

### Using the Chainable API (Recommended)

```javascript
import { AquafierChainable } from 'aqua-js-sdk';

// Prepare your file object
const fileObject = {
  name: "document.pdf",
  hash: "0x...", // File hash
  content: "..." // Optional file content
};

// Create credentials (example for MetaMask)
const credentials = {
  mnemonic: "",      // For CLI signing
  nostr_sk: "",      // For Nostr witnessing
  did_key: "",       // For DID signing
  alchemy_key: "",   // For Ethereum operations
  witness_eth_network: "sepolia",
  witness_method: "metamask"
};

// Create and process your file
const aqua = new AquafierChainable(null);
const result = await aqua
  .notarize(fileObject)
  .sign("metamask", credentials)
  .witness("eth", "sepolia", "metamask", credentials)
  .verify();

// Get results
const tree = result.getValue();
const logs = result.getLogs();
```

### Using the Standard API

```javascript
import { Aquafier } from 'aqua-js-sdk';

const aqua = new Aquafier();

// Create genesis revision
const genesis = await aqua.createGenesisRevision(fileObject);

// Sign the tree
const signed = await aqua.signAquaTree(genesis.data.aquaTree, "metamask", credentials);

// Witness the tree
const witnessed = await aqua.witnessAquaTree(
  signed.data.aquaTree,
  "eth",
  "sepolia",
  "metamask",
  credentials
);

// Verify the tree
const verified = await aqua.verifyAquaTree(witnessed.data.aquaTree, [fileObject]);
```

## SDK Features

### Chainable API

The chainable API provides a fluent interface for building Aqua operations:

```javascript
const result = await aqua
  .notarize(fileObject)           // Create genesis revision
  .sign("metamask", credentials)  // Add signature
  .witness("eth", "sepolia", "metamask", credentials)  // Add witness
  .verify();                     // Verify the complete tree
```

### Standard API

The standard API provides individual methods for each operation:

```javascript
// Step-by-step approach
const genesis = await aqua.createGenesisRevision(fileObject);
const signed = await aqua.signAquaTree(genesis.data.aquaTree, "metamask", credentials);
const witnessed = await aqua.witnessAquaTree(signed.data.aquaTree, "eth", "sepolia", "metamask", credentials);
const verified = await aqua.verifyAquaTree(witnessed.data.aquaTree, [fileObject]);
```

## Error Handling

The SDK provides comprehensive error handling and logging:

```javascript
try {
  const result = await aqua
    .notarize(fileObject)
    .sign("metamask", credentials)
    .witness("eth", "sepolia", "metamask", credentials)
    .verify();
    
  // Success
  const tree = result.getValue();
  const logs = result.getLogs();
  
} catch (error) {
  // Handle errors
  console.error('Aqua operation failed:', error.message);
  
  // Get detailed logs
  const logs = result.getLogs();
  console.log('Operation logs:', logs);
}
```

## Best Practices

<Info>
Follow these best practices for optimal SDK usage:
</Info>

- **Use the chainable API** for simple workflows
- **Handle errors gracefully** with try-catch blocks
- **Validate credentials** before making operations
- **Use appropriate networks** (testnet for development, mainnet for production)
- **Store sensitive credentials securely** (environment variables, secure vaults)
- **Monitor operation logs** for debugging and auditing

## Advanced Usage

### Custom File Objects

```javascript
const customFileObject = {
  name: "contract.sol",
  hash: "0x1234567890abcdef...",
  content: "pragma solidity ^0.8.0; ...",
  metadata: {
    type: "smart-contract",
    version: "1.0.0",
    author: "developer@example.com"
  }
};
```

### Batch Operations

```javascript
// Process multiple files
const files = [file1, file2, file3];
const results = await Promise.all(
  files.map(file => 
    aqua.notarize(file).sign("metamask", credentials).verify()
  )
);
```
